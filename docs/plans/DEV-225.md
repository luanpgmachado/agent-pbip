### Padrao obrigatorio de formatacao JSON

- Numericos decimais:
  - `FORMAT(...)` envolto em `SUBSTITUTE(..., ",", ".")`.
  - enviar como numero JSON quando renderer nao precisar concatenar texto.
  - percentuais na escala exibida: `7.7` para `7,7%`, nao `0.077`.
- Inteiros:
  - enviar sem aspas quando possivel.
  - `FORMAT(..., "0")` so quando inevitavel concatenar como texto.
- Datas:
  - exibicao: `DD/MM` ou `DD/MM/YYYY HH:mm`.
  - ordenacao/serie: `YYYY-MM` ou `YYYY-MM-DD`.
- Valores em branco:
  - string vazia: `""`.
  - numero ausente: `null` quando renderer aceitar; caso contrario `0` apenas para contadores.
  - nao usar `"-"` no JSON novo; fallback visual decide `—`.

### Padrao obrigatorio de escape de string

- Escapar texto dinamico nesta ordem:
  1. barra invertida `\` -> `\\`
  2. aspas duplas `"` -> `\"`
  3. `CR`, `LF` e `TAB` -> espaco simples
- Padrao DAX:

```DAX
VAR Q = UNICHAR ( 34 )
VAR BS = "\"
VAR _txt0 = COALESCE ( [TextoOuColuna], "" )
VAR _txt1 = SUBSTITUTE ( _txt0, BS, BS & BS )
VAR _txt2 = SUBSTITUTE ( _txt1, Q, BS & Q )
VAR _txt3 = SUBSTITUTE ( _txt2, UNICHAR ( 13 ), " " )
VAR _txt4 = SUBSTITUTE ( _txt3, UNICHAR ( 10 ), " " )
VAR _txt5 = SUBSTITUTE ( _txt4, UNICHAR ( 9 ), " " )
```
